## 阅读顺序（推荐）
1. `A_项目整体架构说明.md`：先看全局架构和技术栈。
2. `B_模块依赖图.md`：理解模块边界与入口。
3. `C_核心逻辑地图.md`：看主调用链（Controller -> Service -> Repository）。
4. `D_关键类说明.md`：快速认识关键类职责。
5. `E_设计模式分析.md`：理解实现方法论。
6. `F_复杂代码讲解_ConversationService.md`：深入核心类。
7. `G_项目风险地图.md`：评估并发、性能、重构机会。
